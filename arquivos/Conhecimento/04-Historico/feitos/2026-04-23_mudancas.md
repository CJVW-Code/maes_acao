# Mudanças Realizadas — 23/04/2026
### Mães em Ação · DPE-BA · v3.0 (Draft)

Este documento resume as mudanças realizadas no ciclo de desenvolvimento de hoje, focadas em Segurança de Documentos, Refinamentos de Fluxo e Preparação para o Módulo de BI.

---

## 1. Segurança e Hardening de Downloads [CONCLUÍDO]

Refatoração crítica do fluxo de acesso a documentos para conformidade com LGPD e prevenção de IDOR.

- **Download Tickets (JWT):** Implementado endpoint `POST /:id/gerar-ticket-download` para gerar tokens de curta duração (10 min) específicos para download.
- **Isolamento de Token:** O JWT principal do usuário nunca é exposto na URL de download dos arquivos.
- **Streaming Seguro:** Refatoração do download de ZIP e download individual para validar o ticket e realizar o streaming do Supabase Storage com tratamento de erro.
- **Validação de Purpósito:** O ticket possui binding com o `casoId` e o propósito `download`, impedindo reutilização para outras ações.

---

## 2. Refinamentos de Fluxo e UX [CONCLUÍDO]

Melhorias na estabilidade do sistema e na experiência do defensor/servidor.

### Backend (Geral)
- **Locking Automatizado:** No endpoint `atualizarStatusCaso`, ao mudar para `LIBERADO_PARA_PROTOCOLO`, o sistema agora reseta automaticamente o `servidor_id` e `servidor_at`.
- **Mapping de Casos:** Implementada lógica de fallback inteligente para campos como `{CIDADEASSINATURA}` e `{empregador_nome}` na função `mapCasoRelations`.
- **Prevenção de Regressão:** Corrigida a lógica de `regerarMinuta` para preservar dados de `outros_filhos_detalhes` (exequentes adicionais).

### Frontend (UX)
- **Dashboard Limpo:** Removido o widget de sumário "mini bi" antigo da Dashboard principal para melhorar a performance e clareza visual.
- **Gestão de Cache:** Atualizado `useFormEffects.js` para limpar o `localStorage` após submissão bem-sucedida, eliminando "Zombie Drafts".
- **Formulário de Triagem:** Refatoração do `submissionService.js` para garantir serialização correta de sub-objetos e validações estritas.
